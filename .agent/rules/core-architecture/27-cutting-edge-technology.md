# Cutting-Edge & Stable Technology Mandate (Rule 27)

To ensure VibeOS leverages the latest advancements in software engineering without compromising reliability, all agents (especially **Mr. Pink** during research and scouting phases) must strictly adhere to the following technology selection protocol:

## 1. Proactive Up-to-Date Research
- Whenever researching a solution, proposing an architecture, or finding a package, you **MUST use the web search tools** to find the most current, stable version of the framework, library, or API.
- Do not rely exclusively on internal training data. If a framework has recently released a major update (e.g., Expo SDKs, React, Pydantic V2, latest Python versions), you must read the latest official documentation to leverage new features, syntax, and performance improvements.

## 2. The Stability Threshold
- **Cutting-Edge, Not Bleeding-Edge:** Always prioritize the most recent **stable** release. Do not adopt beta, release-candidate (RC), or alpha versions unless explicitly approved by the CEO or absolutely required by a specific integration.
- Favor highly-starred, actively maintained, and officially supported libraries over niche or abandoned third-party packages.

## 3. Modern Paradigms & Deprecation
- **Modern Syntax:** Always use modern architectural patterns and syntax (e.g., modern React Hooks, async/await properly, modern Python typing features like `|` instead of `Union`).
- **Deprecation Awareness:** Immediately identify and avoid deprecated methods, libraries, or API endpoints. If a dependency is approaching end-of-life, scout and select the modern alternative.

## 4. Specific Agent Duties
- **Mr. Pink (The Scout):** When creating skills or writing `HANDOFF.md` blueprints, you must explicitly dictate the minimum version required (e.g., `FastAPI >= 0.100.0`) and link to the latest official docs to ensure implementation agents use modern syntax.
- **Implementation Agents (Blue, Green, Red, White):** If a blueprint or previous code suggests an outdated method, you are empowered to challenge it, search the web, and implement the modern, stable equivalent.
