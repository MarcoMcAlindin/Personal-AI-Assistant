## Critical Bugs Found

### BUG 1: Double `/v1` in Qwen URL (SEVERITY: CRITICAL)

**Location:** `backend/app/api/v1/endpoints.py`, chat endpoint (line ~175) and email rewrite endpoint (line ~135)

**Problem:** The `QWEN_ENDPOINT_URL` env var is set to `https://vibeos-qwen-599152061719.europe-west1.run.app/v1`. The code then appends `/v1/chat/completions`:
```python
f"{qwen_url}/v1/chat/completions"
```
Resulting in: `https://...run.app/v1/v1/chat/completions` — a 403 Forbidden from the vLLM service.

**Fix:** Change to:
```python
f"{qwen_url.rstrip('/')}/chat/completions"
```
This must be fixed in BOTH the `/chat` endpoint AND the `/email/rewrite` endpoint.

**Note:** The email rewrite endpoint already does `qwen_url.rstrip('/')` on one line but then uses the same broken pattern. The VOS-039 sendoff specifically flagged "Fixed double `/v1` in Qwen URL" as a pre-existing fix — it appears this was only partially applied.

### BUG 2: Auth Bypass Leaks to Unauthenticated Requests (SEVERITY: HIGH)

**Location:** `backend/app/utils/auth.py`

**Problem:** `HTTPBearer(auto_error=False)` means when no `Authorization` header is sent, `credentials` is `None` instead of raising 401. The dev mode check (`VIBEOS_DEV_MODE=true`) runs BEFORE the null check:

```python
def get_current_user(credentials = Depends(_bearer)) -> str:
    if os.environ.get("VIBEOS_DEV_MODE") == "true":
        return os.environ.get("VIBEOS_DEV_USER_ID", "ceo-dev-user")  # <-- runs even with no header
    if credentials is None:
        raise HTTPException(401, ...)  # <-- never reached in dev mode
```

**Impact:** When `VIBEOS_DEV_MODE=true`, ALL protected endpoints are accessible without any auth header whatsoever. This is a security risk even in dev — anyone on the same network can access all API endpoints.

**Fix:** Move the credentials null check BEFORE the dev mode bypass:
```python
def get_current_user(credentials = Depends(_bearer)) -> str:
    if credentials is None:
        if os.environ.get("VIBEOS_DEV_MODE") == "true":
            return os.environ.get("VIBEOS_DEV_USER_ID", "ceo-dev-user")
        raise HTTPException(401, "Missing Authorization header.")

    if os.environ.get("VIBEOS_DEV_MODE") == "true":
        return os.environ.get("VIBEOS_DEV_USER_ID", "ceo-dev-user")
    # ... JWT validation ...
```

Or simpler — just remove `auto_error=False` and keep the original `HTTPBearer()`, which auto-raises 401 when no header is sent. Dev mode bypass then only applies when a Bearer token IS provided (even a fake one).

### BUG 3: Ticketmaster `stateCode=SCT` Returns Zero Results (SEVERITY: HIGH)

**Location:** `backend/app/services/feed_service.py`, `get_concerts()` method

**Problem:** The Ticketmaster Discovery API does not recognize `stateCode=SCT` for Scotland. Live testing confirmed:
- `stateCode=SCT` → 0 events (any genre, any time)
- `city=Glasgow` + `classificationName=music` → 5 events (including Barrowlands, SWG3, O2 Academy)
- `latlong=55.8642,-4.2518&radius=30&unit=miles` + `classificationName=rock,metal` → 10 events

**Fix:** Replace `stateCode=SCT` with geographic search:
```python
params={
    "apikey": api_key,
    "latlong": "55.8642,-4.2518",
    "radius": "50",
    "unit": "miles",
    "classificationName": "rock,metal",
    "size": 20,
    "sort": "date,asc",
}
```

This centers on Glasgow with a 50-mile radius covering Edinburgh, Dundee, and surrounding Scottish venues.

## Additional Observations

### Dev User ID Mismatch
The `.env` has `VIBEOS_DEV_USER_ID=d4f8a1b2-3c4d-5e6f-7890-abcdef123456` but the real user in Supabase is `8b1203a5-4276-475a-a968-185d7536be3f` (`ceo@vibeos.app`). This causes FK constraint violations on INSERT operations (whitelist, tasks) when using the default dev ID. The `.env` should be updated to use the real user UUID.

### Email Inbox Returns Empty
`GET /email/inbox` returns `{"emails": []}`. This is likely because `MOCK_GMAIL` is not set to `"true"` AND no real Gmail OAuth tokens are stored in the user's settings. The mock mode check reads `os.environ.get("MOCK_GMAIL")` — if it's not set, it falls through to real Gmail and silently returns empty on error. Consider setting `MOCK_GMAIL=true` in `.env` for dev, or improve error reporting.

### Code Quality
The implementation closely follows the sendoff blueprints. TaskService, email whitelist CRUD, and the chat save/pin endpoint are all well-structured with proper user_id ownership guards and graceful error handling.
