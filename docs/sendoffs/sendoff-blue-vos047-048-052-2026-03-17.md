## TASK 2 — VOS-048: Mobile vLLM Status Indicator (HIGH)

**Issue:** https://github.com/MarcoMcAlindin/Personal-AI-Assistant/issues/57

**Files:**
- `mobile/src/screens/ChatScreen.jsx`
- `mobile/src/screens/SettingsScreen.jsx`
- `mobile/src/services/api.js`

**What to build:**
Status chip at the top of the Chat screen. Same three-state logic as VOS-047.

Chip behavior:
- `offline` → red chip, "AI Offline", tap triggers warmup POST + transitions to warming
- `warming` → yellow chip, "AI Warming Up...", pulsing animation, tap disabled
- `online` → green chip, "AI Online", informational only

Add to `api.js`:
```js
export async function fetchVllmStatus() {
  const res = await fetch(`${API_BASE_URL}/vllm/status`);
  if (!res.ok) throw new Error('status check failed');
  return res.json(); // { status: 'offline' | 'warming' | 'online' }
}

export async function triggerVllmWarmup() {
  const res = await fetch(`${API_BASE_URL}/vllm/warmup`, { method: 'POST' });
  if (!res.ok) throw new Error('warmup failed');
  return res.json();
}
```

Settings screen: replace any hardcoded server status text with live poll.

Poll every 15 seconds. Clean up on unmount.

## TASK 3 — VOS-052: Native Android Build (HIGH)

**Issue:** https://github.com/MarcoMcAlindin/Personal-AI-Assistant/issues/65

**Goal:** Produce a native APK installable directly on the Samsung Galaxy via ADB. No Expo tunnel. No QR code.

### Step 1 — Run expo prebuild
```bash
cd mobile
npx expo prebuild --platform android --clean
```
This generates the `android/` native project. Commit it (minus secrets).

### Step 2 — Add google-services.json placeholder
Create `mobile/android/app/google-services.json.example` (copy structure, strip real keys). The real file comes from env/secrets and must NOT be committed.

### Step 3 — Verify Health Connect linkage
Open `android/app/build.gradle` and confirm:
- `compileSdkVersion` is 34+
- `react-native-health-connect` appears in the autolinking output

### Step 4 — Add build scripts to `mobile/package.json`
```json
"build:android:debug": "cd android && ./gradlew assembleDebug",
"build:android:release": "cd android && ./gradlew assembleRelease",
"install:android": "adb install -r android/app/build/outputs/apk/debug/app-debug.apk"
```

### Step 5 — Create `mobile/build-android.sh`
Single script: prebuild (if needed) → assembleDebug → adb install.

### Step 6 — Update `.gitignore`
